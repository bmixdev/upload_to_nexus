# Nexus Raw Uploader (Win11, чистый BAT + curl)

## Назначение

Скрипт [`run_uploader.bat`](run_uploader.bat) в консоли запрашивает параметры и выполняет загрузку через `curl`, где можно:

- указать URL raw-репозитория Nexus;
- указать подпуть внутри репозитория;
- указать полный путь к локальному файлу;
- ввести логин и пароль;
- выполнить последовательность **DELETE старого файла** → **UPLOAD нового файла**;
- видеть пошаговый лог прямо в консоли.

## Состав

- [`run_uploader.bat`](run_uploader.bat) — единый сценарий на BAT, выполняющий DELETE/UPLOAD через `curl`.
- [`run_uploader.cfg`](run_uploader.cfg) — файл конфигурации параметров запуска.

## Как использовать

1. Убедитесь, что в системе доступен `curl` (Windows 11 обычно содержит `curl.exe` по умолчанию).
2. При необходимости заполните [`run_uploader.cfg`](run_uploader.cfg) рядом со скриптом.
3. Запустите [`run_uploader.bat`](run_uploader.bat) двойным кликом.
4. Если часть параметров не задана в [`run_uploader.cfg`](run_uploader.cfg), скрипт запросит их интерактивно:
   - **URL репозитория** — например `http://localhost:8081/repository/my-raw`
   - **Подпуть в репозитории** — например `releases/app` (можно пусто)
   - **Локальный файл** — полный путь к файлу
   - **Пользователь Nexus**
   - **Пароль Nexus**
5. Дождитесь выполнения шагов DELETE и UPLOAD.
6. Контролируйте статус по строкам лога в консоли (каждый этап выводится отдельной строкой).

## Формат run_uploader.cfg

Пример содержимого [`run_uploader.cfg`](run_uploader.cfg):

```ini
REPO_URL=http://localhost:8081/repository/my-raw
REPO_PATH=releases/app
FILE_PATH=D:\build\app.zip
NEXUS_USER=deploy
NEXUS_PASS=secret
CONTINUE_ON_DELETE_403_404=1
CURL_INSECURE=1
```

Пояснения:

- `REPO_URL` — базовый URL raw-репозитория;
- `REPO_PATH` — подпуть внутри репозитория (можно пусто);
- `FILE_PATH` — путь к локальному файлу;
- `NEXUS_USER` / `NEXUS_PASS` — учетные данные Nexus;
- `CONTINUE_ON_DELETE_403_404`:
  - `1` — при DELETE `403/404` продолжать UPLOAD;
  - `0` — при DELETE `403/404` останавливать выполнение.
- `CURL_INSECURE`:
  - `1` — `curl` запускается с `-k` и игнорирует ошибки SSL-сертификата;
  - `0` — `curl` выполняет стандартную TLS-проверку сертификата.

## Правила обработки ответов

На этапе DELETE:

- `2xx` — удаление успешно, переход к загрузке;
- `403`, `404` или `405` — логируется предупреждение и загрузка продолжается;
- любой другой код — операция останавливается.

На этапе UPLOAD:

- `2xx` — загрузка завершена успешно;
- любой другой код — вывод ошибки в логе.

## Как собирается целевой URL

Финальный URL артефакта:

`{repoUrl}/{repoPath}/{fileName}`

где:

- `repoUrl` — базовый URL из формы (без конечного `/`);
- `repoPath` — подпуть (без начальных/конечных `/`, может быть пустым);
- `fileName` — имя выбранного локального файла.

Если `repoPath` пуст, получится:

`{repoUrl}/{fileName}`

## Диагностика типовых проблем

- Ошибка `403` на DELETE: обычно недостаточно прав на удаление, но загрузка продолжится по заданному правилу.
- Ошибка `405` на DELETE: метод DELETE может быть ограничен на стороне сервера/прокси; загрузка продолжится по заданному правилу.
- Ошибка `401`/`403` на UPLOAD: проверьте логин/пароль и роли пользователя в Nexus.
- Ошибка «Файл не найден»: проверьте фактический путь, выбранный в форме.
- Ошибки TLS/сертификата: проверьте доступ к Nexus через `curl` на этой машине.
- Неверный HTTP-код `000`: обычно сетевой сбой, блокировка прокси/фаерволом или недоступный URL.

## Учетные данные

- В чистом BAT пароль вводится в консоли видимым текстом (ограничение `set /p`).
- Авторизация в Nexus передается в `curl` через параметр `-u user:password` (Basic Auth).
- Если `NEXUS_PASS` записан в [`run_uploader.cfg`](run_uploader.cfg), пароль хранится в открытом виде в файле.
- По умолчанию в [`run_uploader.cfg`](run_uploader.cfg) включен `CURL_INSECURE=1`, то есть `curl` игнорирует SSL-проверку.

## Ограничения

- Решение ориентировано на Windows 11 и наличие `curl` в `PATH`.
- Решение работает в консоли и не использует `PowerShell`/`HTA`/`ActiveX`.
